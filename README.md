# jalols-page

Personal site for Jaloliddin Ismailov — embedded software · edge AI · climate tech · policy.  
Live at [jalols.page](https://jalols.page).

Stack: plain HTML + CSS + JS. One Node script generates `posts.json` and `feed.xml` from markdown files. GitHub Actions runs it on every push, then deploys to GitHub Pages.

## Structure

```
jalols-page/
├── index.html              # Projects index (card grid + filters)
├── blog.html               # Blog feed (search + sort, loads posts.json)
├── bio.html                # About / CV
├── CNAME                   # Custom domain
│
├── posts/                  # Blog source files — one .md per post
│   ├── _template.md        # Copy this to start a new post
│   └── *.md                # Existing posts (front matter + markdown body)
│
├── posts.json              # Generated by scripts/build.js — do not edit
├── feed.xml                # Generated by scripts/build.js — do not edit
│
├── scripts/
│   └── build.js            # Scans posts/*.md → writes posts.json + feed.xml
│
├── css/
│   └── style.css
│
├── js/
│   ├── nav.js              # Shared nav injector (loaded on every page)
│   ├── main.js             # Code block line numbers + collapse toggle
│   └── projects-data.js    # Project definitions — edit to add projects
│
├── blog/
│   ├── post.html           # Dynamic markdown renderer (serves all .md posts)
│   └── *.html              # Legacy HTML posts (kept for existing links)
│
├── projects/               # One HTML file per project
│
├── img/
│   └── icons/              # Social icons — drop your own SVGs here
│       ├── github.svg
│       ├── linkedin.svg
│       └── email.svg
│
└── .github/
    └── workflows/
        └── deploy.yml      # Runs build.js then deploys on push to main
```

## Writing a blog post

1. Copy `posts/_template.md` to `posts/your-slug.md`.

2. Fill in the front matter at the top:

```markdown
---
title: Your Post Title
date: 2025-06-01
tags: [embedded, edge-ai]
excerpt: One sentence shown on the blog index. Keep it under ~25 words.
---

Post content starts here. Standard markdown — headings, bold, italic,
inline code, fenced code blocks, lists, blockquotes, links, horizontal rules.
```

3. Push. GitHub Actions runs `build.js`, regenerates `posts.json` and `feed.xml`, and deploys.

That's it. No other files to touch.

**Fenced code blocks** get line numbers and a collapse toggle automatically:

````markdown
```python
def hello():
    return "world"
```
````

**Local preview** — run the build script first (blog.html fetches `posts.json` over HTTP):

```sh
node scripts/build.js
python3 -m http.server 8000
# open http://localhost:8000
```

## Adding a project

1. Add an entry to `js/projects-data.js`:

```js
{
  slug:     "my-project",
  title:    "My Project",
  desc:     "One-line description.",
  thumb:    "img/my-project.jpg",
  thumbAlt: "Alt text",
  date:     "2025-06-01",
  status:   "in progress",
  cats:     ["embedded"],           // must match a filter tab: embedded | cloud | paper
  featured: false,
  links:    [{ label: "github", url: "https://github.com/..." }],
},
```

2. Create `projects/my-project.html` — copy `projects/tinyml-wildfire.html` as a template.

## Custom social icons

Drop your SVG files into `img/icons/`:

```
img/icons/github.svg
img/icons/linkedin.svg
img/icons/email.svg
```

Draw them as dark/filled shapes — CSS handles the opacity fade on hover.

## Code annotation markers (HTML posts only)

Inside a code block, annotate a line like this:

```html
<div class="code-line annotated">
  your_code_here();
  <span class="annotation-marker">◆</span>
  <span class="annotation-bubble">Explanation shown on hover.</span>
</div>
```

## Deploying to GitHub Pages

### First-time setup

1. Go to **Settings → Pages** in your GitHub repo.
2. Under *Build and deployment → Source*, select **GitHub Actions**.
3. Push anything to `main` — the workflow fires and the site goes live.

### Every push after that

```sh
git add .
git commit -m "your message"
git push
```

The workflow in `.github/workflows/deploy.yml` automatically:
1. Runs `node scripts/build.js` (regenerates `posts.json` and `feed.xml`)
2. Uploads the full repo as a Pages artifact
3. Deploys

### Custom domain

1. `CNAME` already contains `jalols.page`.
2. Your DNS has `jalols.page → 185.199.x.x` (GitHub Pages IPs) — already configured.
3. Make sure *Enforce HTTPS* is checked in **Settings → Pages**.

## RSS feed

The feed is at `https://jalols.page/feed.xml` and is auto-discovered by browsers and feed readers via `<link rel="alternate">` in the blog pages.
