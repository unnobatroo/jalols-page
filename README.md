# jalols.page

My personal website.

Stack: plain HTML + CSS + JS. One Node script (`scripts/build.js`) scans the
markdown files in `posts/` and generates `posts.json` and `feed.xml`. GitHub
Actions runs it on every push to `main`, then deploys to GitHub Pages.

## Structure

```
jalols-page/
├── index.html              # Home — card grid of all posts + projects, with tag filters
├── post.html               # Renders any posts/*.md by ?slug= (markdown + syntax highlighting)
├── bio.html                # About / CV page
├── CNAME                   # Custom domain (jalols.page)
│
├── posts/                  # Content source — one .md per post or project
│   ├── _template.md        # Copy this to start a new entry
│   └── *.md                # Existing entries (front matter + markdown body)
│
├── posts.json              # Generated by build.js — gitignored, rebuilt by CI (not committed)
├── feed.xml                # Generated by build.js — gitignored, rebuilt by CI (not committed)
│
├── scripts/
│   └── build.js            # Scans posts/*.md → writes posts.json + feed.xml
│
├── assets/                 # All static assets live here
│   ├── css/
│   │   └── style.css
│   ├── js/
│   │   └── nav.js          # Shared top nav, injected on every page
│   └── img/
│       ├── jaloliddin_ismailov_resume.pdf   # Linked from the nav "resume" tab
│       └── icons/                           # Social icons (dark/filled SVGs)
│           ├── github.svg
│           ├── linkedin.svg
│           └── email.svg
│
└── .github/
    └── workflows/
        └── deploy.yml      # Runs build.js then deploys on push to main
```

The home grid builds its tag filters automatically from the tags across all
entries in `posts.json` — add a post with new tags and the filters update on
the next build.

## Writing a post or project

1. Copy `posts/_template.md` to `posts/your-slug.md`.

2. Fill in the front matter:

```markdown
---
title: Your Title
date: 2025-06-01
tags: [embedded, edge-ai]
excerpt: One sentence shown on the index card. Keep it under ~25 words.
type: post          # "post" (default) or "project"
status:             # optional short string shown on project cards
---

Body starts here. Standard markdown — headings, bold, italic, inline code,
fenced code blocks, lists, blockquotes, links, horizontal rules. No `# h1`
needed; the title comes from the front matter.
```

3. Push. GitHub Actions runs `build.js`, regenerates `posts.json` + `feed.xml`,
   and deploys. That's it — no HTML to touch.

Set `type: project` to mark an entry as a project. The card grid and the RSS
feed both read from the generated `posts.json`, so projects and posts live in
the same `posts/` folder.

**Fenced code blocks** get a language label, line numbers, and a collapse
toggle automatically (handled in `post.html`).

## Local preview

`index.html` fetches `posts.json` over HTTP, so run the build first and serve
over a local server:

```sh
node scripts/build.js
python3 -m http.server 8000
# open http://localhost:8000
```

## Resume

The nav "resume" tab links straight to
`assets/img/jaloliddin_ismailov_resume.pdf`. Replace that file to update the
resume (keep the same filename, or update the link in `assets/js/nav.js`).

## Social icons

Drop SVGs into `assets/img/icons/` (`github.svg`, `linkedin.svg`, `email.svg`).
Draw them as dark/filled shapes — CSS handles the opacity fade on hover.

## Deploying to GitHub Pages

### First-time setup

1. **Settings → Pages** in the GitHub repo.
2. Under *Build and deployment → Source*, select **GitHub Actions**.
3. Push to `main` — the workflow fires and the site goes live.

### Every push after that

```sh
git add .
git commit -m "your message"
git push
```

`.github/workflows/deploy.yml` then:
1. Runs `node scripts/build.js` (regenerates `posts.json` + `feed.xml`)
2. Uploads the repo as a Pages artifact
3. Deploys

### Custom domain

1. `CNAME` already contains `jalols.page`.
2. DNS points `jalols.page` → GitHub Pages IPs (already configured).
3. Check *Enforce HTTPS* in **Settings → Pages**.

## RSS feed

The feed at `https://jalols.page/feed.xml` is fully automated — `build.js`
regenerates it from `posts/*.md` on every push (via GitHub Actions), so you
never edit it by hand. Every page links it two ways: a `<link rel="alternate">`
tag in `<head>` (so browsers and feed readers auto-discover it) and a visible
`rss` link in the footer.
