# jalols.page

Stack: plain HTML + CSS + JS. One Node script (`scripts/build.js`) scans the
markdown files in `posts/` and generates `posts.json` and `feed.xml`. GitHub
Actions runs it on every push to `main`, then deploys to GitHub Pages.

## Structure

```
jalols-page/
├── index.html              # Home — card grid with search, tag filters, and sort
├── bio.html                # About / CV page
├── CNAME                   # Custom domain (jalols.page)
│
├── posts/                  # Content source — one .md per post or project
│   ├── _template.md        # Copy this to start a new entry
│   ├── *.md                # Existing entries (front matter + markdown body)
│   └── <slug>/index.html   # Generated page per post — gitignored, rebuilt by CI
│
├── posts.json              # Generated by build.js — gitignored, rebuilt by CI (not committed)
├── feed.xml                # Generated by build.js — gitignored, rebuilt by CI (not committed)
│
├── scripts/
│   └── build.js            # Scans posts/*.md → posts.json, feed.xml, per-post pages
│
├── assets/                 # All static assets live here
│   ├── css/
│   │   └── style.css
│   ├── js/
│   │   ├── nav.js          # Shared top nav, injected on every page
│   │   └── post.js         # Shared post renderer (loaded by every /posts/<slug>/ page)
│   └── img/
│       ├── favicon.svg                     # Site favicon (placeholder)
│       ├── jaloliddin_ismailov_resume.pdf   # Linked from the nav "resume" tab
│       └── icons/                           # Social icons (dark/filled SVGs)
│           ├── github.svg
│           ├── linkedin.svg
│           └── email.svg
│
└── .github/
    └── workflows/
        └── deploy.yml      # Runs build.js then deploys on push to main
```

Each post is served at a clean URL — `/posts/<slug>/` — from a small generated
page (`posts/<slug>/index.html`) that loads the shared renderer
(`assets/js/post.js`). The slug is just the markdown filename.

The home grid builds its tag filters automatically from the tags across all
entries in `posts.json` — add a post with new tags and the filters update on
the next build.

## RSS feed

The feed at `https://jalols.page/feed.xml` is fully automated — `build.js`
regenerates it from `posts/*.md` on every push (via GitHub Actions), so you
never edit it by hand. Every page links it two ways: a `<link rel="alternate">`
tag in `<head>` (so browsers and feed readers auto-discover it) and a visible
`rss` link in the footer.
